# 储能电站日收益计算流程说明

本文档说明本项目从数据生成、优化调度、结算计算到前端展示的完整流程。

---

## 1. 项目概述

本项目模拟一座储能电站在山东电力市场中的日收益结算，核心流程包括：

- **日前市场优化**：基于预测电价制定次日电能量套利计划与 AGC 容量中标方案。
- **实时调度执行**：根据实时 AGC 指令与日前计划，生成功率调度曲线。
- **日度结算**：计算电能量套利收益、调频容量/里程收益、偏差考核、输配电费、运行成本等。
- **前端看板**：通过本地 HTTP 服务器可视化展示收益、功率、SOC、偏差等。

## 3. 数据模型

主要模型定义在 `src/models.py`：

| 模型 | 说明 |
|------|------|
| `StorageParams` | 储能参数：额定功率/容量、SOC 边界、效率、AGC 裕量等 |
| `MarketInterval` | 日前市场单个 15 分钟时段数据 |
| `DayAheadData` | 日前市场 96 点数据 |
| `RealTimeInterval` | 实时运行单个 15 分钟时段数据 |
| `AgcPerformance` | AGC 性能指标 K1/K2/K3/Kp |
| `SettlementParams` | 结算参数：偏差容忍度、惩罚系数、费率等 |
| `IntervalResult` | 单个时段结算结果 |
| `DailyResult` | 每日总结算结果 |

## 4. 完整流程

### 步骤 1：生成示例数据

运行：

```bash
python src/generate_sample_data.py
```

该脚本会：

1. 生成共用的参数文件到 `data/input/`：
   - `storage_params.json`
   - `settlement_params.json`
   - `agc_performance.json`
2. 生成 7 天（默认）的日前市场数据：
   - `data/input/YYYY-MM-DD_day_ahead_market.json`
3. 调用 `optimize_day_ahead()` 生成每日日前中标计划。
4. 调用 `generate_agc_commands()` 生成每日实时 AGC 指令。
5. 调用 `optimize_real_time()` 生成每日实时调度结果，保存到：
   - `data/output/YYYY-MM-DD_real_time_dispatch.json`
6. 调用 `validate_dispatch_constraints()` 校验硬约束。
7. 跨日结转 SOC：下一天的初始 SOC = 上一天估算日末 SOC。

---

### 步骤 2：日前优化 `optimize_day_ahead`

文件：`src/optimizer.py`

输入：

- 日前 LMP 预测电价 `lmp_pred`
- AGC 容量报价 `agc_prices`
- AGC 里程价格 `agc_mileage_price`
- 初始 SOC `initial_soc`

输出：

- `charge_plan`：日前充电计划（96 点，MW）
- `discharge_plan`：日前放电计划（96 点，MW）
- `agc_capacity`：日前 AGC 容量中标（96 点，MW）
- `final_soc`：日末 SOC

优化目标：最大化

```
能量套利收益 + AGC 容量收益 + AGC 里程收益
```

约束：

- SOC 硬边界：`[soc_min, soc_max]`
- SOC 软边界：套利放电不低于 `soc_soft_min`，套利充电不高于 `soc_soft_max`
- 能量功率硬上限：`rated_power_mw - agc_reserve_mw`（永远预留 AGC 功率裕量）
- AGC 容量上限：`agc_max_capacity_mw`

算法：离散 SOC 动态规划（DP）。DP 内部已计入每时段自放电，使规划的 SOC 轨迹与结算端 `simulate_soc` 保持一致。

---

### 步骤 3：实时调度 `optimize_real_time`

文件：`src/optimizer.py`

输入：

- 日前充电/放电计划 `da_charge` / `da_discharge`
- 日前 AGC 容量 `da_agc_cap`
- 实时 LMP `rt_lmp`
- AGC 指令 `agc_commands`
- 初始 SOC `initial_soc`

输出：

- `actual_charge` / `actual_discharge`：实际总充/放电功率
- `arbitrage_charge` / `arbitrage_discharge`：套利部分
- `agc_charge` / `agc_discharge`：AGC 部分
- `actual_agc_mw`：AGC 实际出力
- `agc_mileage`：AGC 实际里程
- `final_soc`：日末 SOC

#### 两段式调度规则

1. **AGC 指令存在时 (AGC != 0)**：
   - 严格按 AGC 指令执行，套利功率 = 0。
   - AGC > 0 时：实际放电 = AGC，实际充电 = 0。
   - AGC < 0 时：实际充电 = |AGC|，实际放电 = 0。
   - 若 AGC 指令会突破 SOC 硬边界，则按物理上限截断，并把 `actual_agc_mw` 记录为实际有效值。

2. **AGC = 0 时**：
   - 在日前中标计划范围内做实时优化。
   - 决策变量：套利充电 `p_charge`、套利放电 `p_discharge`。
   - 硬约束：
     - `0 <= p_charge <= da_charge[t]`
     - `0 <= p_discharge <= da_discharge[t]`
     - 套利充电不得使 SOC 超过 `soc_soft_max`
     - 套利放电不得使 SOC 低于 `soc_soft_min`
   - 优化目标：最大化实时电能量套利收益（按实时 LMP 结算）。
   - 允许不执行满额日前计划（少充/少放），但不允许超出计划。

算法：离散 SOC 动态规划（DP）。AGC 时段只保留“仅响应 AGC”的转移；AGC=0 时段保留套利充/放电转移。

3. **物理安全网**：
   - 输出组装阶段再次按连续 SOC 对 AGC 功率做截断，确保不突破 `[soc_min, soc_max]`。
   - DP 内部已计入每时段自放电，使规划的 SOC 轨迹与结算端 `simulate_soc` 保持一致。

---

### 步骤 4：硬约束校验

文件：`src/validation.py`

校验项：

1. AGC 指令方向/幅值一致，不与套利叠加。
2. AGC = 0 时实际功率不超过日前中标计划（允许在计划内优化导致的少充/少放；允许因 SOC 截断导致的少充/少放）。
3. 不存在“多充电”/“多放电”（实际功率 > 指令）。
4. 不存在同时充电和放电。
5. SOC 不突破硬边界。
6. 实际功率不超过当前 SOC 可执行量。

在 `generate_sample_data.py` 中，每天生成后会自动调用校验。

---

### 步骤 5：日度结算

运行：

```bash
python src/calculator.py
```

或批量：

```bash
python src/calculator.py --all
```

结算流程在 `src/settlement.py` 中调用：

1. `calc_energy_market()`（`src/energy_market.py`）
   - 模拟 SOC 轨迹
   - 计算电能量套利收入
   - 计算偏差考核扣款
   - 计算输配电费、政府基金、电池衰减、辅助用电成本
2. `calc_frequency_market()`（`src/frequency_market.py`）
   - 计算 AGC 容量收益
   - 计算 AGC 里程收益
   - 计算辅助服务分摊
3. 汇总净收益，输出到：
   - `data/output/YYYY-MM-DD_daily_result.json`

跨日结转：第一天使用 `storage_params.json` 中的 `initial_soc`，之后每天使用前一天日末 SOC。

运行：

```bash
python web/run_server.py
```

会自动打开浏览器访问：

```
http://localhost:8080/web/dashboard.html
```

看板页面结构：

1. **收益汇总** + **实时运行数据**（顶部相邻展示）
2. **LMP 电价曲线**
3. **充放电功率 & SOC 曲线**
4. **AGC 收益时序** / **收益构成**
5. **分时段收益分解**
6. 底部保留：日前 vs 实时 LMP 对比、AGC 实际出力、偏差分析、实时数据明细表

## 5. 关键公式

### 电能量套利收益

```
energy_revenue = (actual_discharge - actual_charge) × 0.25h × rt_lmp
```

### 偏差考核

```
charge_excess  = max(0, |actual_charge  - da_charge|  - da_charge  × tolerance)
discharge_excess = max(0, |actual_discharge - da_discharge| - da_discharge × tolerance)
deviation_penalty = (charge_excess + discharge_excess) × 0.25h × rt_lmp × penalty_rate
```

### AGC 容量收益

```
agc_capacity_revenue = win_agc_capacity_mw × agc_capacity_price × (kp / kp_avg) × 0.25h
```

### AGC 里程收益

```
agc_mileage_revenue = actual_agc_mw × agc_mileage_price
```
